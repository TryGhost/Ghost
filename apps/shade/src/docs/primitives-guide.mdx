# Working with primitives

<p className="excerpt">Primitives are how Shade replaces the endless string of `flex flex-col gap-4 items-start` utilities with something a human can read and an AI agent can generate consistently. They give the structural concepts names — `Stack`, `Inline`, `Grid` — so a layout reads like its intent rather than a sequence of Tailwind classes.</p>

## What a primitive is

Most layout code in any React codebase looks the same: a `<div>` with five Tailwind layout utilities. After a while it becomes invisible. Reviewers stop reading those classes; agents glue together their own variants of "vertical stack with medium gap"; small spacing inconsistencies accumulate across the product.

Primitives solve that by giving structural concepts names. Instead of inferring "this is a vertical stack" from `flex flex-col gap-4`, you write `<Stack gap="md">` and the intent is right there. Spacing decisions become semantic (`gap="lg"`) rather than numeric (`gap-6`), so the spacing scale can be tuned without touching every consumer.

## Guidance

- Pick a primitive by what the structure **is**: a column of children → a vertical stack; a row → a horizontal inline; a framed region with padding or radius → a box; a max-width page shell → a container; a two-dimensional layout → a grid; rendered text → the text primitive.
- Don't add anonymous `<div>` wrappers that only carry `flex / grid / gap` utilities. If you need that structure, reach for a primitive instead.
- Use the semantic spacing scale (`gap="lg"`, `padding="sm"`), not raw numbers (`gap-6`, `p-3`). The semantic scale is what the spacing tokens are for.
- Don't put product workflow logic inside a primitive. Primitives are layout vocabulary; if a wrapper is starting to know about Ghost-specific data, it belongs in a Pattern.
- Don't reach for a framing primitive when a stack or inline would do. Framing is the right tool when you genuinely don't want a layout direction.

See the individual primitive entries in the sidebar for the live list of primitives, their props, and runnable examples.

## Adding a new primitive

The bar is high. Primitives are the foundation everything else composes against, and adding one means every reviewer and agent has to learn another name. Before adding one, check that:

- No combination of existing primitives covers the case.
- The structural pattern appears across multiple surfaces, not just the one you're building.
- The name describes structure, not product (`Stack` is good, `MembersList` is not).

When those check out:

1. Add the file at `apps/shade/src/components/primitives/<name>.tsx`.
2. Export it from `apps/shade/src/components/primitives/index.ts`.
3. Add a story at `apps/shade/src/components/primitives/<name>.stories.tsx` with the title `Primitives / <Name>`. The story should cover the props matrix and a few realistic compositions.
4. If the primitive introduces a new token (a spacing step, a tone, a size), add the token to the relevant theme file before using it.

## Examples

A typical page shell composed entirely of primitives:

```tsx
import {Button} from '@tryghost/shade/components';
import {Container, Inline, Stack, Text} from '@tryghost/shade/primitives';

<Container size="page" paddingX="lg">
    <Stack gap="xl">
        <Inline justify="between" align="center">
            <Text as="h1" size="2xl" weight="bold" leading="heading">Members</Text>
            <Button>New member</Button>
        </Inline>

        <Stack gap="md">
            <Text tone="secondary">Manage your audience.</Text>
        </Stack>
    </Stack>
</Container>
```

Read that out loud and it tells you what's there: a page-width container; vertically stacked content; the top row is a horizontal split with the title on the left and the action button on the right; below it, another vertical stack of text.

A list of framed rows:

```tsx
<Stack gap="sm">
    <Text as="h3" size="lg" weight="semibold">Drafts</Text>
    <Grid columns={1} gap="sm">
        <Box padding="md" radius="md" className="border border-border-default">
            <Text>Draft row content</Text>
        </Box>
        <Box padding="md" radius="md" className="border border-border-default">
            <Text>Another draft row</Text>
        </Box>
    </Grid>
</Stack>
```

Migrating Tailwind utility soup to primitives is mechanical and usually shorter:

```tsx
// Before
<div className="flex flex-col gap-4">
    <div className="flex flex-col gap-2">
        <h1 className="text-2xl font-bold">Members</h1>
        <p className="text-gray-600">Manage your audience</p>
    </div>
    <div className="flex items-center justify-between gap-2">...</div>
</div>

// After
<Stack gap="lg">
    <Stack gap="sm">
        <Text as="h1" size="2xl" weight="bold" leading="heading">Members</Text>
        <Text tone="secondary">Manage your audience</Text>
    </Stack>
    <Inline align="center" justify="between" gap="sm">...</Inline>
</Stack>
```
