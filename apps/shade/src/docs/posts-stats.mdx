# Posts–Stats

<p className="excerpt">A temporary parking spot for components shared between the Posts app (<code>apps/posts</code>) and the Stats app (<code>apps/stats</code>) — until those two apps are merged. Not a pattern layer.</p>

## Why this exists

The Posts and Stats apps were split apart by an earlier architectural decision that is being reversed. While they're still separate, some components need to be shared between them, and Shade is the only place they can sit. These components are Ghost-specific to the post-analytics surface — they don't pass the [pattern promotion rules](?path=/docs/overview-architecture--docs) and shouldn't try to.

Today this category holds `PostShareModal` (used by Post Analytics in `apps/posts` and by the Stats overview's "latest post" card in `apps/stats`).

## How to use it

```tsx
import { PostShareModal } from '@tryghost/shade/posts-stats';
```

Import from `@tryghost/shade/posts-stats`, not from `@tryghost/shade/patterns`. The separate entrypoint is deliberate — it makes the temporary status visible at the call site and makes the eventual cleanup mechanical (delete the entrypoint, move the files into the merged app).

## When this category goes away

When the Posts and Stats apps are merged into a single app, this folder and entrypoint disappear and the components move into the merged app. Treat anything in `posts-stats/` as a future migration target, not a candidate for promotion to a real Shade pattern.

## When *not* to add things here

- **A component shared across three or more apps** — that's a real Shade pattern. Apply the [promotion rules](?path=/docs/overview-architecture--docs) and add it under `patterns/` instead.
- **A component shared between two apps that aren't Posts and Stats** — this folder is named for one specific historical situation. Don't generalise it. New cross-app temporary needs get their own named category.
